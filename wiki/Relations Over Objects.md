# Relations Over Objects

Objecthood is often a coordination handle. Meaning often lives in relationships, transformations, constraints, and legal redescription.

Objects are useful at boundaries: APIs, persistence, UI, schemas, protocol edges, documentation, and human-agent coordination. They are suspicious when they become the default interior shape of thought.

## Prototype Pressure

A prototype should represent relationships first and introduce objects only where they help coordination, ownership, persistence, or compilation.

Ask:

- Which objects are semantic?
- Which objects are boundary handles?
- Which objects should be compile-time only?
- Can object structure erase after coordination?

## Failure Signs

- The model starts with nouns before relations are understood.
- Encapsulation hides confusion rather than reducing dependency.
- Runtime object graphs preserve compile-time coordination structure.
- Meaning is recoverable only by inspecting object interactions.
